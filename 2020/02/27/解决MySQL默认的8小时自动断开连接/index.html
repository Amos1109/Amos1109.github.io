解决MySQL默认的8小时自动断开连接

2020-02-27
JSP学习

MySQL服务器默认的“wait_timeout”是28800秒即8小时，意味着如果一个连接的空闲时间超过8个小时，MySQL将自动断开该连接，而连接池却认为该连接还是有效的(因为并未校验连接的有效性)，当应用申请使用该连接时，就会导致上面的报错。

方案一

按异常信息所说，在MYSQL的JDBC连接串上加上autoReconnect=true。但未能解决，据说这是针对MySQL版本是5.0以前的。

方案二

更改配置文件中wait_timeout和interactive_timeout的值。
我先将perfume的数据库文件通过navicat转储到我自己的云服务器上。
网上说修改MySQL安装目录中的配置文件my.ini中的interactive_timeout和wait_timeout，
但是我在配置文件my.ini中没有找到这两个值。可以尝试添加上去。
我就直接在MySQL客户端上改。在MySQL客户端输入命令：
show variables like ‘%timeout%’;
可以看到interactive_timeout = 28800和wait_timeout = 28800，即8小时。
再分别输入set global interactive_timeout=31536000;set global wait_timeout=31536000;
即自动断开时间为1年。但重启MySQL再查询发现，wait_timeout值又变回28800；
通过查阅资料发现，interactive_timeout和wait_timeout的默认值都是28800（8小时）
当这两个参数同时出现时，会以interactive_timeout的值为准。
也就是说不管wait_timeout的值是多少，
用show variables like ‘%timeout%’;查看时显示的两个值都是一样的，
并且都是interactive_timeout的值。而wait_timeout的最大值只允许2147483 （24天左右）。
所以我又重新设置
set global interactive_timeout=2147483;
set global wait_timeout=2147483;
