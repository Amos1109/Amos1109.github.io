hexo基本配置

在文件根目录下的_config.yml，就是整个hexo框架的配置文件了。可以在里面修改大部分的配置。详细可参考官方的配置描述.

参数 描述
title 网站标题
subtitle 网站副标题
description 网站描述
author 你的名字
language 网站语言
timezone 网站时区

发表文章

cd到myblog文件夹下（文件夹内右键也可），执行如下命令新建文章：（[]内文章类型，省略默认为Post)：

$ hexo new [post] articleName

名为articleName.md的文件会建在目录/blog/source/_posts下，articleName是文件名，为方便链接不建议掺杂汉字。

打开目录，最好去下载一个MarkDown编辑器，打开这个.md文件进行编辑。或者使用网络上的在线编辑器。大家需要去学习一下MarkDown编辑器的使用教程，很简单。

文章编辑完成后，cd到myblog文件夹下，执行如下命令来发布：(命令间可以用&&来连接）

hexo clean //清除缓存文件 (db.json) 和已生成的静态文件 (public)
hexo g //生成缓存和静态文件
hexo d //重新部署到服务器

过几分钟就可以在网站上看到你的文章了。

更改主题

你可以到Hexo官网主题页去搜寻自己喜欢的theme。这里以hexo-theme-next为例：NexT主题使用
cd到myblog目录下执行如下命令去下载主题文件：

git clone https://github.com/Sariay/hexo-theme-next.git

将myblog目录下_config.yml里theme的名称landscape修改为next
cd到myblog目录下执行如下命令(每次部署文章的步骤,可以用&&连接步骤，也可以将步骤生成为一个自定义脚本)：

hexo clean //清除缓存文件 (db.json) 和已生成的静态文件 (public)
hexo g //生成缓存和静态文件
hexo d //重新部署到服务器

至于更改theme内容比如名称、描述、头像等去修改myblog/_config.yml文件和myblog/themes/next/_config.yml文件中对应的属性名称即可，而且在NexT 使用文档里有极详细的介绍。

注：Hexo各个风格的主题都有很多强大的功能，比如聊天室，评论，Follow等，希望同学们可以自行去发掘这些功能！
