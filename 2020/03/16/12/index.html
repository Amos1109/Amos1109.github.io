MyBatis简介

MyBatis 是支持定制化 SQL、存储过程以及高级映射的优秀的持久层框架。MyBatis 避免了几乎所有的 JDBC 代码和手动设置参数以及获取结果集。MyBatis 可以对配置和原生Map使用简单的 XML 或注解，将接口和 Java 的 POJOs(Plain Old Java Objects,普通的 Java对象)映射成数据库中的记录。

前期准备

  • 登录https://mybatis.org/mybatis-3/ 阅读MyBatis官方文档，因为有中文版本，对我们很友好，上面把MyBatis的使用方法记载得很全面，上面的一些源码我们也可以直接利用。
  • 登录https://github.com/mybatis/mybatis-3 将MyBatis的jar包下载下来，并引入到相应的Project中，搭建环境。

添加配置

在src目录下，新建sqlMapConfig.xml文件（代码都可以在官方文档里找到）来配置你的数据库连接，例如

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
  PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
  "http://mybatis.org/dtd/mybatis-3-config.dtd">
<configuration>
  <environments default="development">
    <environment id="development">
      <transactionManager type="JDBC"/>
      <dataSource type="POOLED">
        <property name="driver" value="com.mysql.jdbc.Driver"/>
        <property name="url" value="jdbc:mysql://127.0.0.1:3306/hero?useSSL=false"/>
        <property name="username" value="root"/>
        <property name="password" value="994498"/>
      </dataSource>
    </environment>
  </environments>
  <mappers>
    <mapper resource="mapper/UserMapper.xml"/>
  </mappers>
</configuration>

在src目录下，新建一个mapper文件夹，并创建一个Mapper.xml文件，我们可以在官方文档上找到代码格式，直接复制即可。

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
  PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
  "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="org.mybatis.example.BlogMapper">
  <select id="selectBlog" resultType="Blog">
    select * from Blog where id = #{id}
  </select>
</mapper>

在这里我们可以配置一个log4j.properties，jar包在MyBatis中已经存在，不需要重复下载，主要作用就是记录项目运行时产生的动作。比如异常信息，或者关键动作的记录。都可以配置记录到一个文件当中方便查看。关于log4j.properties的一些配置信息可以参考：

https://blog.csdn.net/qq_29166327/article/details/80467593

对数据库的增删改查

每一次对数据库的操作，都需要以下代码的实现：

String resource="sqlMapConfig.xml";
//读取配置文件
InputStream in = Resources.getResourceAsStream(resource);
//需要sqlSessionFactoryBuilder
SqlSessionFactoryBuilder ssfb=new SqlSessionFactoryBuilder();    
//创建sqlSessionFactory
SqlSessionFactory ssf = ssfb.build(in);    
//生产一个sqlSession
SqlSession session = ssf.openSession();

接下来就需要操作数据库的代码实现，比如

增加信息：session.insert("UserMapper.insertUser", user);

此处函数的第一个参数代表要操作的sql语句的路径，这个路径对应Mapper.xml文件里相应的sql语句，第二个参数代表参数，比如这里因为要加一条信息，这个参数就一定是个bean的对象，在加入前我们得先用set函数给这个对象添加全信息。

同时，在Mapper.xml文件添加以下信息：

<!-- 添加用户 -->
  <insert id="insertUser" parameterType="com.hyt.bean.user">
  insert into mistore values(null,#{name},#{price},#{imgPath},#{desc},#{bdesc},#{bimgPath})
  </insert>

其中包括多个参数，比如id，要与路径的信息对应，parameterType指的是传入的参数类型，显然我们这里要传的是user类对象，此外，还有resultType，指输出的参数类型，在查找语句中比较常用，具体的参数请参考官方文档。里面包含的语句和sql语句如出一辙，每一个信息参数我们都用“#｛｝”，里面的名称可以自定义，只要你能看懂。

这是MyBatis的最常用操作，有问题欢迎留言。
