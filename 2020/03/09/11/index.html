数据库连接池的基本思想就是为数据库建⽴⼀个缓冲池，预先向缓冲池中放⼊⼀定数量的连接对象，当
需要获取数据库连接的时候，只需要从缓冲池中取出⼀个对象，⽤完之后再放回到缓冲池中，供下⼀次
请求使⽤，做到了资源的重复利⽤，允许程序重复使⽤⼀个现有的数据库连接对象，⽽不需要重新创
建。

在我看来，JDBC 开发流程大致可分为以下几个步骤，

  1. 加载驱动（只需要加载⼀次）
  2. 建⽴数据库连接（Connection）
  3. 执⾏ SQL 语句（Statement）
  4. ResultSet 接收结果集（查询）
  5. 断开连接，释放资源

C3P0只是将第一步和第二步的代码实现更改了，但是得到的好处并不少。

于是我着手将perfume香水商店的数据库连接更改为C3P0连接池。

1.增加c3p0-config.xml文件

<?xml version="1.0" encoding="UTF-8"?>
<c3p0-config>
<named-config name="testc3p0">
<!-- 指定连接数据源的基本属性 -->
<property name="user">root</property>
<property name="password">994498</property>
<property name="driverClass">com.mysql.jdbc.Driver</property>
<property name="jdbcUrl">jdbc:mysql://121.43.102.125:3306/perfume?useUnicode=true&amp;characterEncoding=UTF8</property>
<!-- 若数据库中连接数不足时, 一次向数据库服务器申请多少个连接 -->
<property name="acquireIncrement">5</property>
<!-- 初始化数据库连接池时连接的数量 -->
<property name="initialPoolSize">20</property>
<!-- 数据库连接池中的最⼩的数据库连接数 -->
<property name="minPoolSize">2</property>
<!-- 数据库连接池中的最⼤的数据库连接数 -->
<property name="maxPoolSize">40</property>
</named-config>
</c3p0-config>

在这里填写一些数据库连接的基本信息，将之前的信息原封不动复制即可。

在这里最重要的一步是去网上下载c3p0所需要的jar包，c3p0-0.9.2.1.jar
然后更改DBHelper文件里面的代码

3.更改itemsDAO里的代码

然后就是更改itemsDAO里的代码，里面的代码主要是进行查询作用的，这里我介绍一个很实用的小工具，叫DBUtils,它将一系列的查询，GET，SET操作进行了封装，这样代码量就大大减少了，就不用写循环代码，一个一个属性添加过去了。主要有以下4种方法：

  • BeanHandler 将结果集映射成 Java 对象 Student
  • BeanListHandler 将结果集映射成 List 集合 List
  • MapHandler 将结果集映射成 Map 对象
  • MapListHandler 将结果集映射成 MapList 结合

使用方法很简单，只需要去网上下载DBUtils的jar包commons-dbutils-1.7.jar就可以了。

其他的任何代码都不需要再修改，就可以直接用了。是不是比之前的代码简练很多呢？
