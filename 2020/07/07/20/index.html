一、功能实现

1.1 首页展示模块

首页支持分类查询、模糊查询以及分页查询。点击查看详情获取对应ID请求到DetailServlet，获取宠物数据。点击加入购物车获取对应ID请求到CartServlet，将Cart对象写入Session。

分页查询使用Cypager插件，以下为前端代码：

CartServlet具有多个操作，以下为代码：

ShoppingCartService shoppingCartService = new ShoppingCartService();
ShoppingCart cart;
//从session中取购物车对象cart
//如果cart为null;第一次请求，新建一个购物车对象
if (request.getSession().getAttribute("cart") == null) {
    cart = new ShoppingCart();
    request.getSession().setAttribute("cart", cart);
} else {
    cart = (ShoppingCart) request.getSession().getAttribute("cart");
}

//获取请求类型get/add/remove/modify
//默认为"get"
String type = "get";
if (request.getParameter("type") != null) {
    type = request.getParameter("type");
}
String id = request.getParameter("id");
String quantity = request.getParameter("quantity");
JsonResult result = new JsonResult();
result.setSuccess(true);
switch (type) {
    case "get":
        request.getRequestDispatcher("cart.jsp").forward(request, response);
        break;
    case "add":
        if (id == null) {
            result.setMsg("商品id参数不正确!");
            result.setSuccess(false);
        }
        if (quantity == null) {
            result.setMsg("商品数量参数不正确!");
            result.setSuccess(false);
        }
        if (result.getSuccess() == true) {
            //调用ShoppingCartService,处理业务
            cart = shoppingCartService.addToCart(Integer.parseInt(id), Integer.parseInt(quantity), cart);
            result.setMsg("加入购物车完成!");
            result.setMsg2(Integer.toString(cart.getTotalCount()));
        }
        break;
    case "remove":
        if (id == null) {
            result.setMsg("商品id参数不正确!");
            result.setSuccess(false);
        }
        if (result.getSuccess() == true) {
            //调用ShoppingCartService,处理业务
            cart = shoppingCartService.removeFromCart(Integer.parseInt(id), cart);
            result.setMsg("删除商品完成!");
            result.setMsg2(Integer.toString(cart.getTotalCount()));
            result.setMsg3(Double.toString(cart.getTotalMoney()));
        }
        break;
    case "modify":
        int quantityInt = Integer.parseInt(quantity);
        int idInt = Integer.parseInt(id);
        if (quantityInt <= 0) {
            result.setMsg("商品数目不得小于等于0件！");
            result.setSuccess(false);
        }
        if (result.getSuccess() == true) {
            cart = shoppingCartService.modifyCart(idInt, quantityInt, cart);
            result.setMsg(Double.toString(cart.getCartItems().get(idInt).getSubTotal()));
            result.setMsg2(Integer.toString(cart.getTotalCount()));
            result.setMsg3(Double.toString(cart.getTotalMoney()));
        }
        break;
}


//输出Json
response.setContentType("text/json;charset=UTF-8");
response.setCharacterEncoding("UTF-8");


PrintWriter out = response.getWriter();
String resultJson = JSON.toJSONString(result);
out.print(resultJson);
out.flush();
out.close();

1.3 后台管理模块

后台管理模块的整体布局以及一些输入表单等均由layui进行开发。后台管理模块包括商店管理（添加宠物，宠物管理，类别管理等），用户管理（用户列表，新增用户等）以及修改密码和安全退出。

在宠物管理内容中，使用了layui的数据表格以及它的分页功能，每条数据均有查看，编辑以及删除功能。数据表格的URL指向PetListServlet， PetListServlet通过传JSON数据到前端，由layui渲染成表格。JS函数中有一个监听工具条函数，监听对数据的删、改、查操作，这些基本的操作都用AJAX实现。

以下是数据表格后端的核心代码：

String pageStr=request.getParameter("page");
String limitStr=request.getParameter("limit");
int page=Integer.parseInt(pageStr);
int limit=Integer.parseInt(limitStr);
CustomerService customerService = new CustomerService();
List<Map<String,Object>> petList=customerService.getPetList(page,limit);
List<PetDTO>pets=new ArrayList<>();
for( int i = 0; i < petList.size(); i++ ) {
    PetDTO pet=new PetDTO(petList.get(i));
    pets.add(pet);
}
int totalRecord=customerService.totalRecord();
String PetJson= JSON.toJSONString(pets);
StringBuffer sb = new StringBuffer();
sb.append("{\"count\":");

sb.append(",\"code\":");
sb.append(0);
sb.append(",\"msg\":");
sb.append("null");
sb.append(",\"data\":");
sb.append(PetJson);
sb.append("}");
//输出Json
response.setContentType("text/json;charset=UTF-8");
response.setCharacterEncoding("UTF-8");

PrintWriter out = response.getWriter();
out.print(sb.toString());
out.flush();
out.close();

删除操作：删除操作比较简单，直接获取宠物id提交到Servlet，进行数据库删除操作，然后JSON回调成功函数后，在前端用layui相应的函数把对应的那一行删去。

修改操作：我又新增了一个petEdit.jsp页面，其中只包括一个表单，点击修改操作后，通过layui弹出层获取对应宠物id请求到PetEditServlet获取宠物信息，然后转发弹出这个jsp页面，便可显示宠物信息并对其进行修改，提交到UpdatePetServlet，让数据库更新。其中在上传宠物图片功能上，我用了layui的图片上传功能，并使用了两个Servlet，在选择图片后，会自动发送数据到ImageUploadServlet，对图片进行随机UUID重名以及上传到云服务器上，并通过JSON回传图片名到前端，前端渲染出图片，图片上传成功。然后再按保存按钮，获取新上传的图片名称以及其他修改的内容，提交数据到UpdatePetServlet，让数据库更新。这样才算修改了一只宠物的图片。

二、技术总结

AJAX：能够异步请求到Servlet并回传数据到前端，更新页面的部分元素，比单纯的表单Post的应用性强，扩展性高。在整个项目开发中，大概80%的请求servlet都用到了AJAX，因为确实比较好用。

FastJson：阿里出品的转JSON字符串的工具，因为前端页面大量用到JSON字符串，用手自己拼装字符串比较繁琐，就用了这个小工具，可以把各式各样的对象通过函数转换成JSON字符串，但在回传日期数据需要在对应的实体类对象的日期属性添加@JSONField(format=”yyyy-MM-dd HH:mm:ss”)的注解，不然会传过来一串数字。

JSTL标签：在jsp页面中使用标签的形式完成业务逻辑等功能。jstl出现的目的同el一样也是要代替jsp页面中的脚本代码。使得页面更加整洁。

Bootstrap:对于一个前端菜鸟，是做不出好看的html页面的，Bootstrap框架提供了优雅的HTML和CSS规范，可以做出比较好看、简洁的html页面，商店除了后台的所有页面都用了Bootstrap框架。

Cypager分页：这是一个我在github上找到的前端分页工具，在前端使用起来比较简单，但是在后端要有对应的逻辑函数去支持它，在后端得建立一个pager的工具类，其中包括pageSize(每页显示条数)，currentPage(当前页)，totalRecord(总记录条数)，totalPage(总页数)以及petList(数据列表)，在数据表查询后进行逻辑运算获得这些数据并回给前端，进行分页。

Layui：又是一个前端菜鸟的救星，用它可以做出优秀的后台管理页面，非常支持前后端交互的操作。用它做出的数据表格的分页操作比Cypager简单，在后端只要几行代码就能搞定。其中的弹出层，表单初始化、图片上传功能等都非常好用。layui 提供了丰富的内置模块，他们皆可通过模块化的方式按需加载.

Dbutils：Java与数据库的连接包括：导包、注册驱动、获取与数据库的连接对象、获取SQL语句的执行者对象、获取结果集对象、关闭连接等，但DBUtils简化了JDBC的开发步骤，使得我们可以用更少量的代码实现连接数据库的功能。

jQuery:toastr、ajax以及元素选择器等，基本写JS函数都要以jQuery为基础，来写更少的代码，做更多的事情。它封装JavaScript常用的功能代码，提供一种简便的JavaScript设计模式，优化HTML文档操作、事件处理、动画设计和Ajax交互。

MVC设计模式：MVC 是一种使用 MVC（Model View Controller 模型-视图-控制器）设计创建 Web 应用程序的模式：

Model（模型）表示应用程序核心（比如数据库记录列表）。

View（视图）显示数据（数据库记录）。

Controller（控制器）处理输入（写入数据库记录）。

PetStore运用JSP（view） + servlet(Controller) + javabean(Model)的设计模式，将前端和后端分别开来，更有利于代码的设计与维护。

三、技术完善

3.1 对类别、用户等信息的增删改查需要进一步添加；

3.2 没有增加使用Filter过滤器，网页安全性能不高；

3.3 分页查询点击页码后需要去重新查询数据库，效率比较低，最好一次性查询到所有数据后保存，分页时直接调用；

3.4 没有使用MD5加密密码，安全性不高；

3.5 在前端页面的分类查询以及添加宠物时的分类选择，暂时没有从数据库查找类别内容，代码耦合度不高。
